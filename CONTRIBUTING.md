# Contributing to Kontiki

External contributions are possible, but Kontiki is maintained according to its maintainer’s priorities and available time.

For any non-trivial change, please open an issue first and explain:

- the problem
- why the change belongs in Kontiki
- the intended scope

Contributions are more likely to be considered when they are:

- small and focused
- aligned with the framework’s design
- documented and tested when relevant

Proposals that significantly expand the scope, reshape the architecture, or increase maintenance cost may remain outside the project.

## Project model

Kontiki is a maintainer-led project.

Its technical direction, public API evolution, and release decisions are coordinated by its maintainer.

The project evolves on available time and does not follow a fixed public schedule.

By submitting a contribution, you agree that it is provided under the same license as the project.
